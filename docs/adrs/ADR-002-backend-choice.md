# ADR 002: Lựa chọn Django cho Backend
## Trạng thái
Accepted
## Bối cảnh
Hệ thống cần một công nghệ backend mạnh mẽ, tốc độ phát triển phù hợp với quy mô team 4 người, dễ triển khai.
## Quyết định
Sử dụng Django cho Backend
## Lý do
1. Tốc độ phát triển nhanh
2. Có nhiều tài liệu và cộng đồng hỗ trợ
3. Tích hợp sẵn nhiều tính năng như ORM, authentication
## Hệ quả
1. Tích cực: 
+ Ổn định và lâu dài
+ Dễ mở rộng và nâng cấp
+ Bảo mật tốt
2. Tiêu cực:
+ Tốc độ xử lý có thể chậm hơn so với các framework khác
